Privacy

This page describes how the chinmeister web dashboard (chinmeister.com) handles data when you open it from the CLI or sign in with a token.

What you send. The dashboard sends your bearer token to the chinmeister API to load teams, agents, shared memory, and coordination state for projects where you are a member. Tokens are issued when you run npx chinmeister init (or chinmeister init after npm install -g chinmeister) or log in via the CLI; they are stored in ~/.chinmeister/config.json on your machine.

Browser storage. If you use “connect with a token”, the dashboard may keep the token in sessionStorage for the duration of the browser tab so you stay signed in until you sign out or close the tab.

Where data lives. Project and account data are processed and stored by the chinmeister backend (Cloudflare Workers / Durable Objects) as described in the product architecture. We do not sell personal data.

Security. Report vulnerabilities per the project SECURITY.md on GitHub.

Changes. This notice may be updated as the service evolves. The dashboard link here is the current reference.
