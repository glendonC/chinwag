02 / Memory

Shared project context

Keep useful project knowledge alive across sessions.

When one agent finds a doc, design, gotcha, or project-specific rule, the next agent should start there instead of rediscovering it from scratch.

Shared memory captured once, reused later
How it works
  1. 01

    An agent finds something worth keeping: a doc, a decision, a design reference, or a project-specific gotcha.

  2. 02

    chinwag saves it as shared project knowledge instead of letting it disappear with the session.

  3. 03

    The next agent starts with that context available, even across tools and handoffs.

03 / Privacy

Clear boundaries

Useful coordination without shipping your code.

chinwag coordinates around file paths, short task summaries, and project knowledge you choose to retain. Your source files stay on your machine.

Boundary map what is shared, what is not
Never leaves your machine
  • Source code and file contents
  • Secrets, keys, and tokens
  • Local conversation and git history
Shared only for coordination
  • Which files agents are touching
  • Short task summaries
  • Project knowledge you choose to retain
Why that matters

chinwag is valuable because it gives agents shared awareness and memory across sessions, not because it ingests your codebase.
